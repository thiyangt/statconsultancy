Dear Ms.~Dassanayake,

We would be truly grateful if you could follow the below given set of guidelines when compiling the final data set. The below guidelines will aid you to compile your data into a \emph{tidy data} format which is extremely helpful to understand the structure of the dataset and extract the required variables being more time efficient by reducing the excessive time spent on data cleaning.

The standard criteria to follow for the overall data structure is as follows:

\begin{itemize}
\tightlist
\item
  Every column is a variable
\item
  Every row is an observation
\item
  Every cell is a single value
\end{itemize}

All the guidelines mentioned below will help you to compile the final data set adhering to the above criteria.

\hypertarget{structure-of-data-set-worksheets}{%
\section{Structure of data set (worksheets)}\label{structure-of-data-set-worksheets}}

The data set can be compiled into two worksheets namely;

1.1. Company specific data and;

1.2. Macro-economic data.

Please refer the format given below when entering the data.

\textbf{1.1 Company (Firm) specific data}

The more convenient format out of the following two formats can be chosen when entering the firm specific data:

\emph{1. Entering all the data from 2007-2018 of a single firm at once followed by the other firms.}

\emph{2. Entering all the data of all the firms for a particular year at once followed by the other years.}

\textbf{1.2. Macro-economic data}

\hypertarget{additional-guidelines}{%
\section{Additional Guidelines}\label{additional-guidelines}}

Additionally, please kindly follow the below specifications when entering the data.

\begin{itemize}
\tightlist
\item
  \textbf{Avoid using colors:} Do not color the fonts or the cells which can make the data set visually noisy
\item
  \textbf{Be uniform:} Use the same format and units for same type of data \emph{e.g: If capital letters are used for the name of a firm, use capital letters for all the other names of firms, if a certain observation of a variable is in percentage format make sure all values of the variable follows that format.}
\item
  \textbf{Use abbreviations:} To avoid the variable names being too long, use meaningful abbreviations which can be then further described through the data dictionary. \emph{Please refer section on data dictionary for more information.}
\end{itemize}

\hypertarget{data-dictionary}{%
\section{Data Dictionary}\label{data-dictionary}}

A data dictionary is a descriptions of variables and their definitions and properties in a given data set. They are very useful to give an overall idea about the data set to a data analyst and hence enabling much more meaningful interpretations of results.

The following tabular format can be followed when creating a simple data dictionary.

\begin{longtable}[]{@{}lcrr@{}}
\toprule
Abbreviation & Variable & Description & Notes \\
\midrule
\endhead
TDER & Total Debt to Equity Ratio & Xxx & Xxx \\
STDR & Short Term Debt to Equity & Xxx & Xxx \\
\bottomrule
\end{longtable}

\emph{Note: The above abbreviations are used as basic examples of creating the data dictionary. Please do feel free to use any such meaningful abbreviation}

\begin{itemize}
\tightlist
\item
  \textbf{Abbreviation:} The variable name of a given column
\item
  \textbf{Variable:} The expanded name of the variable (column)
\item
  \textbf{Description:} A brief description about the variable, ensure to mention the specific formula used if many forms of formulas can be used to obtain the same measure
\item
  \textbf{Notes:} Any additional information with regards to the observations of the variable that might be important when analyzing the data
\end{itemize}
